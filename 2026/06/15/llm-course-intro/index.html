《LLM Deep Learning》—— 一门面向训练和推理工程师的代码优先 LLM 课程

这门课是什么

LLM Deep Learning 是一门 code-first 的 LLM 实战课,核心定位是"为训练和推理工程师而写":不堆概念、不追新模型版本,而是用 Python + PyTorch 从零实现 LLM 的每一个核心组件,然后把这些组件装配成一个能训、生成、对齐、部署、评估的完整工程系统。

课程的副标题很直白:

Starting from code, 11 chapters to build a complete large language model engineering system.

11 章 / 251 个 section(章下小节),覆盖从 BPE tokenization 到 vLLM 级推理优化的完整链路,每个 chapter 配套编程练习 + 概念题 + 工业级架构案例。

五大核心特点

1. Code-first,每一行都自己写

跟很多"读论文+看代码"的课程不同,本课的学习循环是:

深度理论 → 理解"为什么" → 编程练习(你写代码) → 对照参考实现 → 概念题巩固

每个 chapter 起手就是"Prerequisites / In-Chapter Internalization / Validation Signal"三段,把数学、PyTorch、统计、系统知识按章节任务就地展开,不另开 side track。数学/ML 基础只在你需要时再补,附录(math-prerequisites.html)是参考,不必先修。

2. 工业级架构案例,不是玩具实现

课程不只教你写一个能 forward 的 GPT,而是用 MLA、MoE、FP8、GRPO/DAPO/GSPO、sparse/compressed attention、linear attention、SSM/Mamba-style layers、learnable residual connections 这一串工业级设计,让你看清每个组件背后的工程瓶颈,而不是追某个模型版本的规格表。

3. "概念 + 编程"双练习,完成度可验证

每个 chapter 都配套: - 编程作业 starter.py + reference_solution.py + tests.py(11 个 assignments/chXX_xxx/) - 概念/手算题 - 明确的 "Completion Criteria"(你能解释清什么就算学完)

例如 Ch01 BPE 的完成标准是"能解释一段文本如何变成 token ID,token ID 如何还原成文本"。

4. 知识图谱化,前后章强关联

每章顶部都有SVG 知识地图:标出"学这章前需要知道什么 / 这章解决什么 / 学完后接着读什么"。

5. 配套 9 份工程附录,工程严谨度高

docs/ 下有 9 份独立但互引的文档: - math-prerequisites.html — 数学/PyTorch 符号约定、tensor flow、mask 语义、反向传播 - ml-foundations-prerequisite-bridge.html — 从 ML 基础到本课的桥接 - python-pytorch-review-session.html — Python/PyTorch 速查 - reading-list.html — 扩展阅读 - written-problem-set.html / worked-example-pack.html — 配套习题 - classic-nlp-handout.html / classic-nlp-deep-dive-module.html — RNN/LSTM/seq2seq/BERT 经典 NLP 衔接

这是大学级课程质量的关键:每个知识点都回答三件事——

  1. 数学上:它解决什么问题,公式里每个量是什么
  2. 代码上:怎么落到 PyTorch 的 tensor、shape、mask、dtype、gradient
  3. 工程上:引入了什么 cost/speed/quality/stability 的取舍

三阶段学习路径(课程自带的初学者指南)

课程体量大、覆盖到工业前沿,第一次学不要把高级话题全当必修。课程建议三遍走:

阶段 章节 目标 建议
第一遍:主干必修 Ch01-Ch06 把文本变成 token,搭一个能 forward 的 GPT 必须写代码作业;DeepSeek 扩展只需理解动机
第二遍:让它跑起来 Ch07-Ch08 训练一个小模型,让它能生成文本 重点看 loss/optimizer/sampling 的输入输出 shape
第三遍:进阶选读 Ch09-Ch10 微调、对齐、RAG、推理优化、前沿架构 先建立概念地图,再回头补公式和工程细节

最低前置:会 Python 函数/列表/字典、基础矩阵乘法、知道 PyTorch tensor 的 shape。数学推导最初吃力时,先抓住"输入是什么、输出是什么、shape 怎么变"就够了。

如何使用

1. 浏览(零安装)

直接打开 http://garry-x.github.io/llm-course 即可阅读全部 11 章 + 9 份附录。

2. 本地起服务(推荐)

打开 http://127.0.0.1:8080 即可。serve.sh 会自动检测端口占用并询问是否终止占用的进程。

3. 跑作业(需要 Python 环境)

requirements.txt 极简(只有 4 个核心包):

每个作业(assignments/chXX_xxx/)有 4 个文件:

  • README.md — 任务说明
  • starter.py — 你的代码起点
  • reference_solution.py — 参考实现
  • tests.py — 验证脚本

跑测试统一用仓库根的 run_assignment_tests.py。

4. 课程使用建议

  • 不要跳章:Ch01-Ch06 的链式结构,每一节都建立在前一节的张量流上
  • 三遍走:先 Ch01-Ch06 主干 → 再 Ch07-Ch08 跑通 → 最后 Ch09-Ch10 选读
  • 做笔记:每章的"Validation Signal"是自检清单,你能复述就说明学到位
  • 看附录:遇到符号/shape/mask 模糊时,翻 docs/math-prerequisites.html

适合谁

  • ✅ 懂 Python + 基础线性代数,想从代码层真正"造一个 LLM"
  • ✅ 已经会用 transformers / LangChain,但想搞清里面到底怎么算的
  • ✅ 做训练 / 推理工程,需要把 MLA、MoE、KV Cache、FlashAttention、GRPO 这些词从"听过"变成"能写能调"
  • ❌ 不适合"只想调 API 调 prompt"的纯应用层
  • ❌ 不适合完全没写过 Python 的纯数学背景

总结

这门课的稀缺之处在于"代码深度 + 工程视野"双拉满:既不像纯论文课只讲公式,也不像纯项目课只跑 demo。每个 chapter 都有"数学 / 代码 / 工程"三栏自检,251 个 section + 11 个作业 + 9 份附录,形成了一个自洽的工程学习闭环。

如果你是认真想搞懂 LLM 训练和推理全链路的工程师,推荐从 Ch01 BPE 开始,把 assignments/ch01_bpe/starter.py 写完,跑通 tests.py,然后决定要不要继续。
