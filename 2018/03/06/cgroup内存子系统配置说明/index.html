cgroup内存子系统配置说明

2018-03-06
Linux
container

基于 Linux kernel 3.10

内存子系统

参考：kernel/Documentation/cgroups/memory.txt

配置项详细参数：

配置项 说明
tasks 绑定进程/线程; 显示当前关联进程/线程
cgroup.procs 显示当前关联进程
cgroup.event_control eventfd接口
memory.usage_in_bytes 显示当前内存使用量
memory.limit_in_bytes 设置/显示内存使用量硬限，内存使用不会超过该限制
memory.failcnt 内存使用超限次数
memory.max_usage_in_bytes 显示记录的最大内存量
memory.soft_limit_in_bytes 设置/显示内存使用量软限，内存使用可超过该限制。内存压力较高时，将回收内存至软限制值
memory.stat 显示详细统计信息
memory.use_hierarchy 设置/显示是否统计子group内存
memory.force_empty 清空group使用内存，使用时需保证Group内没有进程/线程
memory.pressure_level 与cgroup.event_control配合，设置内存压力通知
memory.swappiness 设置/显示内存换出权重
memory.move_charge_at_immigrate 控制/显示进程迁移到其他group后，原有内存统计如何迁移
memory.oom_control 关闭/打开oom killer; 显示当前oom信息； 设置oom通知
memory.numa_stat 显示每个numa节点的内存使用情况
memory.kmem.* 限制内核内存使用
  • 对于memory.memsw.*配置项，memsw代表该配置应用于内存+Swap，不在上表中赘述
  • 内存统计、限制均包含使用的Page Cache内存
  • 配置内存压力通知
    • 使用eventfd(2)接口创建eventfd
    • open memory.pressure_level文件
    • 将”<event_fd> <memory.pressure_level的fd> <压力级别>” 写入cgroup.event_control
      • 压力级别有critical、low、medium、三种，critical表示将要oom，或者oom killer已经运行；low表示当前内存使用比较紧张，内存分配会触发内存回收；medium表示当前有适中的内存压力，系统可能正在执行内存页换出。
  • 设置move_charge_at_immigrate 为0，进程迁移到其他group，已分配内存仍占用原group统计，原有group删除后，内存统计由其父group或者root group接管。
  • 设置oom通知的方法与设置内存压力通知类似。
